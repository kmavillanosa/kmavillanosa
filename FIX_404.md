# Fix 404 Error - Step by Step Guide

## The Problem
You're seeing a 404 error because GitHub Pages either:
1. Isn't enabled yet
2. Files haven't been pushed to GitHub
3. Deployment hasn't completed

## Solution - Follow These Steps

### Step 1: Commit and Push Your Files

Make sure all files are committed and pushed to GitHub:

```bash
# Check status
git status

# Add all files
git add .

# Commit
git commit -m "Setup headless CMS for GitHub Pages"

# Push to GitHub
git push origin main
```

**Important**: Wait for the push to complete before proceeding.

### Step 2: Enable GitHub Pages

1. **Go to your repository settings**:
   - Visit: https://github.com/kmavillanosa/kmavillanosa/settings/pages

2. **Configure Pages**:
   - Under **"Source"**, you have two options:

   **Option A: GitHub Actions (Recommended)**
   - Select: **"GitHub Actions"** as the source
   - This will use the workflow file we created (`.github/workflows/pages.yml`)
   - Save the settings

   **Option B: Deploy from a branch**
   - Select: **"Deploy from a branch"**
   - **Branch**: Select `main`
   - **Folder**: Select `/ (root)`
   - Click **Save**

3. **Wait for deployment**:
   - Go to: https://github.com/kmavillanosa/kmavillanosa/actions
   - Look for "Deploy to GitHub Pages" workflow
   - Wait until you see a green checkmark (✅)
   - This usually takes 1-3 minutes

### Step 3: Verify Deployment

1. **Test the root page**:
   - Visit: https://kmavillanosa.github.io/
   - You should see your homepage

2. **Test the test page**:
   - Visit: https://kmavillanosa.github.io/test.html
   - If this works, GitHub Pages is enabled correctly

3. **Test the CMS**:
   - Visit: https://kmavillanosa.github.io/cms/admin/index.html
   - This should now work!

### Step 4: If Still Getting 404

**Check these:**

1. **Verify files are in the repository**:
   - Go to: https://github.com/kmavillanosa/kmavillanosa
   - Check that `index.html` exists in the root
   - Check that `cms/admin/index.html` exists

2. **Check branch name**:
   - Make sure you're using `main` branch (not `master`)
   - If your default branch is `master`, update the workflow file

3. **Wait longer**:
   - GitHub Pages can take 5-10 minutes to propagate
   - Clear browser cache (Ctrl+Shift+Delete)
   - Try incognito/private mode

4. **Check Actions tab**:
   - Go to: https://github.com/kmavillanosa/kmavillanosa/actions
   - Look for any failed workflows
   - Click on a workflow to see error messages

### Step 5: Verify File Structure

Your repository should have this structure:
```
kmavillanosa/
├── .github/
│   └── workflows/
│       └── pages.yml
├── .nojekyll
├── index.html
├── test.html
├── cms/
│   ├── admin/
│   │   ├── index.html
│   │   └── config.yml
│   ├── content/
│   └── media/
└── README.md
```

## Quick Diagnostic Commands

Run these in your terminal to verify:

```bash
# Check if files exist locally
ls -la index.html
ls -la cms/admin/index.html
ls -la .nojekyll

# Check git status
git status

# Check if you're on main branch
git branch
```

## Still Not Working?

1. **Check GitHub Status**: https://www.githubstatus.com/
2. **Verify repository is public** (or you have GitHub Pro for private repos)
3. **Try accessing**: https://kmavillanosa.github.io/test.html first
4. **Check browser console** (F12) for any errors

## Expected Timeline

- **Push to GitHub**: Immediate
- **GitHub Pages enable**: 1-2 minutes
- **First deployment**: 2-5 minutes
- **Full propagation**: Up to 10 minutes

**Total wait time**: Usually 5-10 minutes from enabling Pages to full availability.
